Sobre o projeto

Esta aplicação é o resultado de um trabalho para a disciplina de Programação para a Web, ministrada por Hudson Borges, na FACOM, UFMS. Ela foi pensada e desenvolvida como uma ferramenta simples e eficiente para o seu propósito. Acreditamos ter alcançado esse objetivo, e sem abrir mão da segurança e beleza de uma aplicação web moderna.

Algumas características:

Foi utilizado o padrão MVC, uma vez que o flask foi pensado para funcionar dessa forma. Os fomulários possuem validadores de dados, tais como de email, campos obrigatórios, tamanho mínimo e máximo. Os fomulários de cadastro e login possuem verificação de usuário já existente, senha incorreta, avisando ao usuário especificamente o que aconteceu durante uma falha de cadastro ou login. Usuários não conseguem acessar, modificar, nem excluir listas e tarefas de outros usuários. Cadastre-se agora mesmo e teste nossa aplicação!

Sobre as ferramentas

Front-end

Foi utilizado, majoritariamente, o framework Bootstrap 5.0.1. Também implementamos estilos próprios de acordo com a necessidade, mas também evitando a repetição de classes já trazidas pelo framework utilizado. A parte principal da aplicação (gerenciar listas e tarefas) segue um conceito de SPA, evitando grandes cargas de processamento do lado do servidor. Para tornar isso possível, utilizamos jQuery, e muito Javascript. Para requisições assíncronas, utilizamos ajax.

Back-end

Foi utilizado o micro-framework flask, na linguagem Python versão 3.9. A vantagem da utilização deste micro-framework, é que a aplicação fica mais leve, devido ao fato de que não há uma carga desnecessária de módulos que não são utilizados. Os módulos utilizados foram: Flask-WTF, para manuseio de formulários. Flask-Login, para gerenciamento de login. Flask-Bcrypt, para encriptar senhas. Peewee, para mapeamento objeto-relacional. Vale destacar que a template engine utilizada pelo flask é o Jinja2, e a WSGI é implementada pela biblioteca Werkzeug. O banco de dados utilizado foi o SQLite3, devido a ausência da necessidade de um servidor. Também, como a aplicação não exige um banco robusto, esta foi a melhor opção.
